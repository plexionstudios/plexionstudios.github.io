Void Rising Datapack

Supporting 1.13-1.16, this datapack makes the void rise from y=0 to y=245, making the game more of a challenge.

Description

A lightweight data-pack for Minecraft Java Edition 1.16 that well, makes the void rise. Everything is controlled within the single datapack with customisable variables for timings if needed. Lag is handled well with falling blocks being killed to avoid sand blocks causing lag upon rising. Upon dying, you are automatically placed into spectator mode to allow you to watch the game in progress without interference.

Installation

As with every datapack, there is a very simple installation process.

  1. Click the download button above then save it in your Downloads.
  2. Open your singleplayer menu, press Create New World then Data Packs
  3. Drag the .zip file from your Downloads into the available space on the left.
  4. Click the right arrow button and move it over (ignore any version errors - it works fine)
Setup

Once the datapack is installed, you can start the void rising with some simple set-up.

  1. Teleport to 0 0 (x and z) in your world.
  2. When you want the void to begin rising, type function voidrising:start
  3. If you want to pause (or stop) the void rising, you can set the enabled score in the global scoreboard to 0.

You can also configure timings for the warning in warning_time and the actual rising itself in rising_time (measured in ticks - 20 ticks = 1 second)

Updates
Version 2.1Published 18/01/2021

With this update you can now disable some of the pre-built features and enable some new ones.

  • Automatic spectator mode upon death is now an option you can disable with the spectator_death score in the global scoreboard.
  • Automatic stopping of rising upon y245 is now an option you can disable with the height_limit score in the global scoreboard.
  • You can now disable all sound effects with the sound_effects score in the global scoreboard.
  • The actionbar messages (The void has risen etc.) can now be disabled with the rising_status score in the global scoreboard.
  • You can replace the actionbar with a new bossbar that displays the progress until the void rises by changing the bossbar_status score in the global scoreboard.
Version 2.0Published 17/01/2021

This update fixes a few issues and adds some quality of life changes. Here we go:

  • The void now stops rising at y245 (can be changed inside the datapack)
  • Timings for the warning & rising event have been sped up
  • The variables names have been changed from time1 and time2 to warning_time and rising_time
  • When the void rises, the actionbar now displays it's current y level
